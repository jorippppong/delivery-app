# 기술 면접 질문과 답변

## Java 기본 개념

### 제네릭(Generic)

- **질문**: 제네릭이란 무엇인가요?
- **답변**: 타입을 매개변수로 사용하여 컴파일 시점에 타입 안정성을 보장하고 코드 재사용성을 높이는 기능입니다

### 컬렉션 프레임워크

- **질문**: ArrayList와 LinkedList의 차이점은 무엇인가요?
- **답변**: ArrayList는 배열 기반으로 랜덤 접근이 빠르고, LinkedList는 노드 기반으로 삽입/삭제가 빠릅니다

### 예외 처리

- **질문**: Checked Exception과 Unchecked Exception의 차이점은 무엇인가요?
- **답변**: Checked Exception은 컴파일 시점에 처리해야 하고, Unchecked Exception은 런타임에 발생하는 예외입니다

### 예외 관리

- **질문**: 예외 관리를 어떻게 할 것인가요?
- **답변**: GlobalExceptionHandler를 통해 중앙 집중식으로 예외를 처리하고, 커스텀 예외 클래스를 만들어 비즈니스 로직별로 구분하여 관리합니다

### 공통 관리

- **질문**: 공통 관리를 어떻게 할 것인가요?
- **답변**: AOP를 활용해 로깅, 보안, 트랜잭션 등을 공통으로 처리하고, 공통 유틸리티 클래스와 상수 클래스를 만들어 코드 중복을 방지합니다

### 인터페이스 vs 추상클래스

- **질문**: 인터페이스와 추상클래스의 차이점은 무엇인가요?
- **답변**: 인터페이스는 다중 상속이 가능하고 메서드만 선언하며, 추상클래스는 단일 상속이고 구현체를 가질 수 있습니다

### 오버로딩 vs 오버라이딩

- **질문**: 오버로딩과 오버라이딩의 차이점은 무엇인가요?
- **답변**: 오버로딩은 같은 이름의 메서드를 매개변수만 다르게 정의하는 것이고, 오버라이딩은 상위 클래스의 메서드를 재정의하는 것입니다

### 람다식

- **질문**: 람다식이란 무엇인가요?
- **답변**: 함수형 프로그래밍을 지원하는 익명 함수로, 코드를 간결하게 작성할 수 있게 해주는 기능입니다

### 스트림 API

- **질문**: 스트림 API란 무엇인가요?
- **답변**: 컬렉션 데이터를 함수형 스타일로 처리할 수 있게 해주는 API로, 필터링, 매핑, 집계 등의 연산을 지원합니다

### 어노테이션

- **질문**: 어노테이션이란 무엇인가요?
- **답변**: 코드에 메타데이터를 추가하는 기능으로, 컴파일러나 프레임워크가 특별한 처리를 할 수 있게 해줍니다

### 가비지 컬렉션

- **질문**: 가비지 컬렉션이란 무엇인가요?
- **답변**: 더 이상 사용되지 않는 메모리를 자동으로 해제하는 Java의 메모리 관리 기법입니다

### 메모리 영역

- **질문**: JVM 메모리 영역에 대해 설명해주세요
- **답변**: 힙(객체 저장), 스택(메서드 호출), 메서드 영역(클래스 정보), PC 레지스터, 네이티브 메서드 스택으로 구성됩니다

### String vs StringBuffer vs StringBuilder

- **질문**: String, StringBuffer, StringBuilder의 차이점은 무엇인가요?
- **답변**: String은 불변 객체이고, StringBuffer는 동기화된 가변 객체이며, StringBuilder는 동기화되지 않은 가변 객체입니다

### equals vs ==

- **질문**: equals와 ==의 차이점은 무엇인가요?
- **답변**: ==는 참조 비교이고, equals는 내용 비교입니다

### final 키워드

- **질문**: final 키워드의 용도는 무엇인가요?
- **답변**: 변수는 상수화, 메서드는 오버라이딩 방지, 클래스는 상속 방지의 용도로 사용됩니다

### static 키워드

- **질문**: static 키워드의 특징은 무엇인가요?
- **답변**: 클래스 레벨에서 공유되며, 인스턴스 생성 없이 접근 가능하고 메모리에 한 번만 로드됩니다

### 접근 제어자

- **질문**: 접근 제어자에 대해 설명해주세요
- **답변**: public(모든 곳), protected(같은 패키지+상속), default(같은 패키지), private(같은 클래스)로 접근 범위를 제한합니다

## 객체지향 프로그래밍

### 객체 추상화 확장성

- **질문**: 객체 추상화와 확장성에 대해 설명해주세요
- **답변**: 인터페이스 기반 설계로 구현체 교체가 용이하고, 의존성 주입으로 결합도를 낮춰 확장성을 확보합니다

### 의존성 역전 원칙

- **질문**: 의존성 역전 원칙이 무엇인가요?
- **답변**: 고수준 모듈이 저수준 모듈에 의존하지 않고, 둘 다 추상화(인터페이스)에 의존하는 원칙입니다

### SOLID 원칙

- **질문**: SOLID 원칙에 대해 설명해주세요
- **답변**: SRP(단일 책임), OCP(개방-폐쇄), LSP(리스코프 치환), ISP(인터페이스 분리), DIP(의존성 역전) 원칙입니다

### 디자인 패턴

- **질문**: 싱글톤 패턴이 무엇인가요?
- **답변**: 클래스의 인스턴스를 하나만 생성하여 전역적으로 접근할 수 있게 하는 디자인 패턴입니다

### 팩토리 패턴

- **질문**: 팩토리 패턴의 목적은 무엇인가요?
- **답변**: 객체 생성 로직을 별도로 분리하여 코드의 유연성과 확장성을 높이는 패턴입니다

## 비동기 프로그래밍

### 동기 vs 비동기

- **질문**: 동기와 비동기의 차이점을 설명해주세요
- **답변**: 동기는 순차 실행(블로킹), 비동기는 병렬 실행(논블로킹)으로 성능 향상을 위해 사용합니다

### 스레드와 프로세스

- **질문**: 스레드와 프로세스의 차이점은 무엇인가요?
- **답변**: 프로세스는 독립적인 실행 단위이고, 스레드는 프로세스 내에서 실행되는 경량 실행 단위입니다

### 가비지 컬렉션

- **질문**: 가비지 컬렉션이란 무엇인가요?
- **답변**: 더 이상 사용되지 않는 메모리를 자동으로 해제하는 Java의 메모리 관리 기법입니다

## API 개발

### API 문서 작성

- **질문**: API 문서를 코드 안에 어떻게 작성하나요?
- **답변**: Swagger/OpenAPI 3.0의 @Operation, @Parameter 어노테이션을 사용해 코드와 문서를 동기화합니다

### RESTful API

- **질문**: RESTful API란 무엇인가요?
- **답변**: REST 아키텍처 스타일을 준수하는 API로, HTTP 메서드와 URI를 활용하여 자원을 관리합니다

### HTTP vs HTTPS

- **질문**: HTTP와 HTTPS의 차이점은 무엇인가요?
- **답변**: HTTPS는 HTTP에 SSL/TLS 암호화를 추가하여 데이터 전송의 보안을 강화한 프로토콜입니다

## 데이터베이스

### N+1 문제

- **질문**: N+1 문제를 어떻게 해결하나요?
- **답변**: JOIN FETCH나 @EntityGraph를 사용해 한 번의 쿼리로 연관 데이터를 함께 조회합니다

### 영속성과 생명주기

- **질문**: JPA 엔티티의 생명주기에 대해 설명해주세요
- **답변**: Transient(비영속) → Persistent(영속) → Detached(준영속) → Removed(삭제) 상태로 변화하며, 영속 상태에서 변경 감지가 일어납니다

### 순환참조

- **질문**: 순환참조 문제를 어떻게 해결하나요?
- **답변**: @JsonIgnore 사용하거나 DTO를 활용해 JSON 직렬화 시 무한 루프를 방지합니다

### SQL vs NoSQL

- **질문**: SQL과 NoSQL의 차이점은 무엇인가요?
- **답변**: SQL은 관계형 데이터베이스로 정형 데이터를, NoSQL은 비관계형 데이터베이스로 비정형 데이터를 주로 다룹니다

### 정규화

- **질문**: 데이터베이스 정규화에 대해 설명해주세요
- **답변**: 중복 데이터를 제거하고 데이터 무결성을 보장하기 위해 테이블을 분리하는 과정입니다

### 인덱스

- **질문**: 데이터베이스 인덱스의 역할은 무엇인가요?
- **답변**: 검색 성능을 향상시키기 위해 특정 컬럼에 대한 정렬된 데이터 구조를 만드는 것입니다

### 트랜잭션

- **질문**: 트랜잭션의 ACID 특성에 대해 설명해주세요
- **답변**: 원자성(Atomicity), 일관성(Consistency), 격리성(Isolation), 지속성(Durability)으로 데이터 무결성을 보장합니다

## Spring Framework

### Spring Boot

- **질문**: Spring Boot의 장점은 무엇인가요?
- **답변**: 자동 설정과 내장 서버로 빠른 개발이 가능하고, 스타터 의존성으로 간편한 설정이 가능합니다

### 의존성 주입

- **질문**: 의존성 주입이 무엇인가요?
- **답변**: 객체 간의 의존성을 외부에서 주입받아 결합도를 낮추고 테스트 용이성을 높이는 디자인 패턴입니다

### MVC 패턴

- **질문**: MVC 패턴이란 무엇인가요?
- **답변**: 모델(Model), 뷰(View), 컨트롤러(Controller)로 구성된 소프트웨어 디자인 패턴입니다

### ORM

- **질문**: ORM이란 무엇인가요?
- **답변**: 객체 지향 프로그래밍 언어의 객체와 관계형 데이터베이스의 데이터를 매핑하는 기술입니다

## 보안

### JWT

- **질문**: JWT 토큰이 무엇인가요?
- **답변**: JSON 형태로 사용자 정보를 담은 토큰으로, 서버에서 검증 가능한 디지털 서명이 포함된 인증 방식입니다

### SQL Injection

- **질문**: SQL Injection을 어떻게 방지하나요?
- **답변**: PreparedStatement 사용이나 JPA의 파라미터 바인딩을 통해 사용자 입력을 안전하게 처리합니다

### XSS

- **질문**: XSS 공격이란 무엇인가요?
- **답변**: 웹사이트에 악성 스크립트를 삽입하여 사용자 정보를 탈취하는 공격 기법입니다

### CSRF

- **질문**: CSRF 공격을 어떻게 방지하나요?
- **답변**: CSRF 토큰을 사용하거나 SameSite 쿠키 속성을 설정하여 방지합니다

### OAuth

- **질문**: OAuth란 무엇인가요?
- **답변**: 타사 애플리케이션이 사용자 정보를 접근할 수 있도록 권한을 부여하는 표준 프로토콜입니다

## 성능 최적화

### 캐싱

- **질문**: 캐싱의 목적은 무엇인가요?
- **답변**: 자주 사용되는 데이터를 메모리에 저장해 데이터베이스 접근을 줄이고 응답 속도를 향상시킵니다

### 로드 밸런서

- **질문**: 로드 밸런서란 무엇인가요?
- **답변**: 여러 서버에 트래픽을 분산시켜 부하를 균형 있게 분배하는 장치나 소프트웨어입니다

### CDN

- **질문**: CDN이란 무엇인가요?
- **답변**: 콘텐츠를 사용자와 가까운 서버에 캐싱하여 빠르게 제공하는 분산 네트워크입니다

### 스케일 업 vs 스케일 아웃

- **질문**: 스케일 업과 스케일 아웃의 차이점은 무엇인가요?
- **답변**: 스케일 업은 기존 서버의 성능을 향상시키는 것이고, 스케일 아웃은 서버 대수를 늘려 성능을 향상시키는 것입니다

## 개발 방법론

### TDD

- **질문**: TDD란 무엇인가요?
- **답변**: 테스트를 먼저 작성하고 그에 맞는 코드를 구현하는 개발 방법론입니다

### CI/CD

- **질문**: CI/CD란 무엇인가요?
- **답변**: 지속적 통합(Continuous Integration)과 지속적 배포(Continuous Deployment)를 통해 소프트웨어 개발의 효율성을 높이는 방법론입니다

### 마이크로서비스

- **질문**: 마이크로서비스 아키텍처란 무엇인가요?
- **답변**: 애플리케이션을 작은 독립적인 서비스들로 분리하여 개발, 배포, 확장성을 높이는 아키텍처입니다

### DDD

- **질문**: DDD란 무엇인가요?
- **답변**: 도메인 모델을 중심으로 소프트웨어를 설계하고 개발하는 방법론입니다

## 인프라/DevOps

### Docker

- **질문**: Docker란 무엇인가요?
- **답변**: 애플리케이션을 컨테이너로 패키징하여 배포하고 실행할 수 있게 해주는 플랫폼입니다

### Kubernetes

- **질문**: Kubernetes란 무엇인가요?
- **답변**: 컨테이너화된 애플리케이션의 배포, 스케일링, 관리를 자동화하는 오픈소스 플랫폼입니다

### API 게이트웨이

- **질문**: API 게이트웨이란 무엇인가요?
- **답변**: 여러 API 서비스의 엔드포인트를 단일 진입점으로 통합하여 관리하는 서버입니다

### 클라우드 컴퓨팅

- **질문**: 클라우드 컴퓨팅이란 무엇인가요?
- **답변**: 인터넷을 통해 컴퓨팅 자원(서버, 스토리지 등)을 제공하는 서비스입니다

## 네트워크

### TCP vs UDP

- **질문**: TCP와 UDP의 차이점은 무엇인가요?
- **답변**: TCP는 연결 지향적이고 신뢰성이 높으며, UDP는 비연결 지향적이고 속도가 빠르지만 신뢰성이 낮습니다

### DNS

- **질문**: DNS란 무엇인가요?
- **답변**: 도메인 이름을 IP 주소로 변환해주는 시스템입니다

### CORS

- **질문**: CORS란 무엇인가요?
- **답변**: 웹 브라우저에서 다른 도메인의 리소스에 접근할 때 발생하는 보안 정책으로, 이를 우회하기 위한 메커니즘입니다

### 웹소켓

- **질문**: 웹소켓이란 무엇인가요?
- **답변**: 클라이언트와 서버 간에 양방향 통신을 가능하게 하는 프로토콜입니다

## 기타

### 쿠키 vs 세션

- **질문**: 쿠키와 세션의 차이점은 무엇인가요?
- **답변**: 쿠키는 클라이언트에 저장되고, 세션은 서버에 저장되는 사용자 상태 정보입니다

### Git

- **질문**: Git과 SVN의 차이점은 무엇인가요?
- **답변**: Git은 분산 버전 관리 시스템이고, SVN은 중앙 집중식 버전 관리 시스템입니다

### 리팩토링

- **질문**: 리팩토링이란 무엇인가요?
- **답변**: 코드의 기능은 유지하면서 구조를 개선하여 가독성과 유지보수성을 높이는 작업입니다

### 메모리 누수

- **질문**: 메모리 누수란 무엇인가요?
- **답변**: 프로그램이 더 이상 필요하지 않은 메모리를 해제하지 않아 발생하는 메모리 낭비 현상입니다

## 추가 질문 모음 - OOP / Database / Spring

### 객체지향 (추가)

- **질문**: 캡슐화의 목적은 무엇인가요?
- **답변**: 변경 가능 지점을 외부로부터 숨겨 결합도를 낮추고 응집도를 높입니다

- **질문**: 다형성이란 무엇이고 왜 유용한가요?
- **답변**: 동일 인터페이스에 다양한 구현을 바인딩해 확장성과 테스트 용이성을 높입니다

- **질문**: 상속보다 합성을 선호하는 이유는?
- **답변**: 런타임 조합이 가능해 결합도를 낮추고 재사용성을 높입니다

- **질문**: 불변 객체(Immutable)를 쓰는 이유는?
- **답변**: 스레드 안전성과 예측 가능성을 제공해 버그를 줄입니다

- **질문**: 값 객체(Value Object) vs 엔티티(Entity)?
- **답변**: 값 객체는 동일성보다 동등성이 중요, 엔티티는 식별자 기반 동일성이 중요합니다

- **질문**: LSP 위반의 예는?
- **답변**: 하위 타입이 상위 타입 계약을 깨면 대체 가능성이 무너집니다 (ex. 사각형/정사각형)

- **질문**: 인터페이스 분리 원칙(ISP) 적용 방법은?
- **답변**: 클라이언트별 최소 인터페이스로 분리해 불필요한 의존을 제거합니다

- **질문**: OCP를 실무에서 어떻게 지키나요?
- **답변**: 전략/팩토리/정책 매핑으로 새 기능은 추가만 하고 기존 코드는 수정하지 않습니다

### 데이터베이스 (추가)

- **질문**: 인덱스 자료구조는 무엇을 쓰나요?
- **답변**: 대부분 B+Tree, 동등 조건 특화는 Hash 인덱스를 사용합니다

- **질문**: 커버링 인덱스란?
- **답변**: 필요한 컬럼이 모두 인덱스에 있어 테이블 접근을 생략하는 인덱스입니다

- **질문**: 실행 계획에서 주로 보는 지표는?
- **답변**: 접근 방식(Full Scan/Index Range), 조인 순서/방식, 비용, 필터 적용 시점입니다

- **질문**: 조인 종류와 차이?
- **답변**: NL/Hash/Merge 조인으로, 데이터 분포·정렬·메모리에 따라 선택합니다

- **질문**: 트랜잭션 격리 수준 차이는?
- **답변**: Read Uncommitted/Committed, Repeatable Read, Serializable로 부정합 허용 범위가 다릅니다

- **질문**: MVCC는 무엇을 해결하나요?
- **답변**: 잠금 경쟁 없이 일관된 읽기를 제공해 동시성을 높입니다

- **질문**: 파티셔닝 vs 샤딩?
- **답변**: 파티셔닝은 단일 DB 내부 분할, 샤딩은 여러 DB 인스턴스로 수평 분산입니다

- **질문**: 정규화 vs 반정규화 선택 기준은?
- **답변**: 쓰기 무결성은 정규화, 읽기 성능/리포팅은 반정규화를 고려합니다

- **질문**: 페이징 성능 개선 방법은?
- **답변**: 커서 기반, 인덱스 힌트, covering index, keyset pagination을 사용합니다

### Spring Framework (추가)

- **질문**: IoC/DI는 무엇이며 장점은?
- **답변**: 객체 생성/주입을 컨테이너가 맡아 결합도를 낮추고 테스트를 용이하게 합니다

- **질문**: 빈(Bean) 스코프 종류는?
- **답변**: singleton, prototype, request, session, application 등이 있습니다

- **질문**: 빈 생명주기 콜백은?
- **답변**: 초기화/소멸 메서드, @PostConstruct/@PreDestroy, BeanPostProcessor 등입니다

- **질문**: @Transactional 주의사항은?
- **답변**: public 메서드, 프록시 자기호출 회피, 체크드 예외 롤백 설정에 유의합니다

- **질문**: 트랜잭션 전파 레벨 예시는?
- **답변**: REQUIRED(기본), REQUIRES_NEW(새 트랜잭션), MANDATORY 등 용도에 맞게 선택합니다

- **질문**: AOP를 어디에 쓰나요?
- **답변**: 로깅, 모니터링, 트랜잭션, 보안 같은 횡단 관심사를 모듈화합니다

- **질문**: 필터 vs 인터셉터 vs AOP 차이?
- **답변**: 서블릿 레벨/스프링 MVC 레벨/프록시 기반으로 적용 범위와 시점이 다릅니다

- **질문**: RestTemplate vs WebClient 차이?
- **답변**: RestTemplate은 동기, WebClient는 비동기/리액티브로 고성능에 적합합니다

- **질문**: @Controller vs @RestController 차이?
- **답변**: @RestController는 @ResponseBody가 결합돼 기본적으로 JSON 응답을 반환합니다

- **질문**: 검증(Validation)은 어떻게 구성하나요?
- **답변**: Bean Validation(@Valid, 제약 애노테이션)과 전역 예외 처리로 표준화합니다

- **질문**: 캐싱은 어떻게 적용하나요?
- **답변**: @Cacheable/@CacheEvict로 읽기 성능을 높이고 캐시 무효화 전략을 설계합니다

- **질문**: Security 필터 체인의 역할은?
- **답변**: 인증/인가 과정을 단계별 필터로 구성해 요청을 보호합니다
